<details>
<summary><b>🔷 Retail POS Billing & Inventory Management System</b> — Distributed Enterprise Retail Platform</summary>

<br/>

> A high-throughput, multi-tenant B2B retail platform engineered for enterprise-scale operations — built on a production-grade Microservices Architecture using ASP.NET Core Web API, RabbitMQ-driven asynchronous messaging, and JWT-secured role-based access control.

<div align="center">

| Dimension | Detail |
|:---|:---|
| **Stack** | ASP.NET Core Web API · RabbitMQ · PostgreSQL · JWT · Microservices |
| **Scale** | Multi-tenant B2B · Concurrent storefront synchronization |
| **Performance** | Asynchronous messaging layer for real-time inventory sync across decoupled viewports |
| **Security** | Stateless JWT session management · Role-based merchant admin authorization |
| **Architecture** | Modular microservices · DB schema isolation · Concurrency conflict resolution |
| **Impact** | Eliminated checkout race-conditions · Scalable enterprise retail backbone |
| **Repository** | [![GitHub](https://img.shields.io/badge/View%20Repository-181717?style=flat-square&logo=github&logoColor=white)](https://github.com/jithendra98) |

</div>

Architected from the ground up as a modular, horizontally scalable distributed system. Each service operates as an independently deployable unit with isolated database schemas to prevent transactional conflicts. The RabbitMQ messaging layer decouples inventory events from storefront consumers, ensuring eventual consistency without tight coupling. Role-based JWT authorization provides granular access segmentation across merchant admin panels without server-side session state.

| **UX** | Fully responsive Streamlit interface · Low-friction interaction layer |
| **Impact** | Production-ready AI research assistant with enterprise retrieval accuracy |

The Streamlit front-end provides a production-clean interface with minimal cognitive overhead.
